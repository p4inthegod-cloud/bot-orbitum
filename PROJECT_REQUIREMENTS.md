- Риск-менеджмент
- Журнал сделок

### Урок 2: Постоянство в трейдинге (8 материалов)

- 4 фактора избежания нестабильности
- Торговля без ошибок
- Трейдинг и вероятности
- 5 фундаментальных рыночных истин
- 4 типа страха трейдера
- Заключение

### Урок 3-4: Трендовая структура (10 материалов)
- Как правильно смотреть на концепты

- Свинги
- Основа и слом структуры
- HTF и LTF структура
- Premium/Discount
- ОТЕ
- Вход в сделку
- Таймфреймы
- Моментум

### Урок 5-6: Ликвидность (8 материалов)
- Ликвидность в классическом понимании

- Как это применять?
- Примеры сделок после снятия ликвидности
- Ордерфлоу
- Новости

### Урок 7-11: Точки интереса (5 материалов)
- Imbalance (FVG)
- Sell to Buy / Buy to Sell (S2B/B2S)

### Урок 12-13: Ренджи (3 материала)

- Дополнение к точкам интереса: SMT

### Урок 15: Сессии (1 материал)
- Торговые сессии

### Урок 16: Проп-трейдинговые компании (2 материала)
- Prop Trading Firms
- Психология при прохождении отбора

**Итого: 56 материалов обучения**

## 🎯 Функциональные Требования Бота

### 1. Воронка Продаж (Funnel)
- **Первый контакт**: Приветственное сообщение с описанием обучения
- **Демо-доступ**: Бесплатный доступ к первым 3-5 материалам (Урок 1)
- **Upgrade**: Кнопка для покупки полного доступа
- **Платежная система**: Интеграция с платежной системой (Stripe/YooKassa)

### 2. Управление Подписками
- **Бесплатная подписка**: Доступ к ограниченному контенту
- **Премиум подписка**: Полный доступ ко всем 56 материалам
- **Статусы пользователей**: 
  - `free` - бесплатный доступ
  - `premium` - полный доступ
  - `trial` - пробный период

### 3. Система Доставки Контента
- **Структурированное меню**: Навигация по урокам и материалам
- **Прямые ссылки**: На материалы в Notion или загруженные PDF/видео
- **Прогресс отслеживания**: Какие материалы пользователь уже просмотрел
- **Закладки**: Возможность сохранять избранные материалы

### 4. Система Рассылок
- **Автоматические рассылки**: 
  - Приветствие при регистрации
  - Напоминания о новых материалах
  - Еженедельные дайджесты
- **Ручные рассылки** (от администратора):
  - Отправка сообщений выбранным пользователям
  - Рассылка по статусу подписки (free/premium)
  - Отправка от имени бота

### 5. Админ-панель
- **Управление пользователями**:
  - Просмотр списка пользователей
  - Изменение статуса подписки
  - Блокировка/разблокировка
- **Управление контентом**:
  - Добавление новых материалов
  - Редактирование описаний
  - Управление доступом (free/premium)
- **Рассылки**:
  - Создание и отправка сообщений
  - Планирование рассылок
  - Статистика доставки
- **Аналитика**:
  - Количество пользователей
  - Статистика подписок
  - Активность пользователей

### 6. Персональные Сообщения
- **Возможность писать от имени бота** конкретным пользователям
- **История переписки** в админ-панели
- **Автоматические ответы** на часто задаваемые вопросы

## 💾 Модель Данных

### User
```
- id: Telegram ID
- username: Telegram username
- first_name: Имя
- last_name: Фамилия
- subscription_status: free | premium | trial
- subscription_expires: DateTime (для trial)
- joined_at: DateTime
- last_active: DateTime
- is_blocked: Boolean
- viewed_materials: [material_ids]
- bookmarked_materials: [material_ids]
```

### Material
```
- id: UUID
- lesson_id: Номер урока (1-16)
- title: Название материала
- description: Описание
- content_type: text | video | pdf | link
- content_url: URL контента
- access_level: free | premium
- order: Порядок отображения
- created_at: DateTime
```

### Subscription
```
- id: UUID
- user_id: Telegram ID
- status: free | premium | trial
- plan_type: monthly | yearly | lifetime
- price: Цена в рублях
- payment_id: ID платежа
- expires_at: DateTime
- created_at: DateTime
```

### Broadcast
```
- id: UUID
- title: Название рассылки
- content: Текст сообщения
- target_users: all | free | premium | specific_ids
- scheduled_at: DateTime (если запланирована)
- sent_at: DateTime
- status: draft | scheduled | sent
- recipients_count: Количество получателей
- created_by: Admin ID
```

## 🔧 Технический Стек

### Backend
- **Node.js + Express** или **Python + FastAPI**
- **Database**: PostgreSQL или MongoDB
- **Telegram Bot API**: node-telegram-bot-api или python-telegram-bot
- **Платежи**: Stripe API или YooKassa API
- **Кеширование**: Redis (опционально)

### Frontend (Админ-панель)
- **React + TypeScript** (уже инициализирован)
- **Tailwind CSS + shadcn/ui**
- **API клиент**: Axios/Fetch

### Хостинг
- **Backend**: Railway, Render, или Heroku
- **Database**: PostgreSQL на том же хостинге
- **Frontend**: Manus или Vercel

## 📋 Фазы Разработки

1. **Backend API** (Node.js + Express + PostgreSQL)
   - Endpoints для управления пользователями
   - Endpoints для управления контентом
   - Endpoints для рассылок
   - Интеграция платежей

2. **Telegram Bot** (node-telegram-bot-api)
   - Обработчики команд
   - Меню навигации
   - Система доставки контента
   - Обработка платежей

3. **Админ-панель** (React)
   - Dashboard с аналитикой
   - Управление пользователями
   - Управление контентом
   - Система рассылок
   - Персональные сообщения

4. **Интеграции**
   - Платежная система
   - Уведомления
   - Логирование

## 🚀 MVP (Минимальный Жизнеспособный Продукт)

Для быстрого запуска:
1. Telegram бот с меню навигации
2. Бесплатный доступ к первым 3 материалам
3. Простая админ-панель для отправки рассылок
4. Базовая система управления подписками
5. Интеграция с платежной системой (Stripe)
